# راهنمای نصب Mirza Bot — از صفر تا اولین /start

این سند برای کسی است که **پنل صنعتی (Marzban / 3x-ui / …)** را دارد و می‌خواهد **ربات فروش VPN** را روی **سرور + تلگرام** راه بیندازد.  
نسخهٔ کد شما در این مخزن: **0.1.5** (با بهبودهای امنیتی اخیر). اسکریپت رسمی `install.sh` نسخهٔ Pro را از گیت‌هاب نصب می‌کند؛ مراحل تلگرام در هر دو حالت تقریباً یکی است.

---

## قبل از شروع — چه چیزهایی لازم است؟

| مورد | توضیح |
|------|--------|
| سرور | ترجیحاً **Ubuntu 22** با دسترسی root |
| دامنه | یک زیردامنه مثل `bot.example.com` که با **A record** به IP سرور اشاره کند |
| SSL | گواهی معتبر (Let's Encrypt) — تلگرام **فقط HTTPS** برای webhook قبول می‌کند |
| PHP + MySQL | PHP 8.x، افزونه‌های `pdo_mysql`, `curl`, `mbstring`, `json` |
| تلگرام | یک اکانت برای ساخت ربات و یک اکانت برای شما (ادمین) |

---

## بخش ۱ — کارهایی که **فقط در تلگرام** انجام می‌دهید

### گام ۱: ساخت ربات با BotFather

1. در تلگرام به **`@BotFather`** بروید و `/start` بزنید.
2. دستور **`/newbot`** را بفرستید.
3. یک **نام نمایشی** برای ربات انتخاب کنید (مثلاً `فروشگاه VPN من`).
4. یک **یوزرنیم** انتخاب کنید که حتماً به **`bot`** ختم شود (مثلاً `MyShopVPN_bot`).
5. BotFather یک **توکن** می‌دهد؛ شبیه این است:

   ```
   7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

   این همان **`APIKEY`** در `config.php` است. **هرگز** آن را در گروه یا گیت‌هاب نگذارید.

6. (اختیاری ولی توصیه‌شده) در BotFather:
   - **`/setdescription`** — توضیح کوتاه برای کاربران
   - **`/setabouttext`** — درباره ربات
   - **`/setuserpic`** — آواتار
   - **`/setcommands`** — لیست دستورات (بعد از نصب می‌توانید از پنل ادمین ربات هم تنظیم شود)

### گام ۲: پیدا کردن **Chat ID** خودتان (ادمین اصلی)

ربات باید بداند «شما» کی هستید تا دسترسی ادمین بدهد.

**روش ساده:**

1. به رباتی که ساختید (مثلاً `@MyShopVPN_bot`) بروید و **`/start`** بزنید (حتی قبل از نصب سرور هم اشکالی ندارد).
2. در مرورگر این آدرس را باز کنید (`TOKEN` را با توکن واقعی عوض کنید):

   ```
   https://api.telegram.org/botTOKEN/getUpdates
   ```

3. در JSON دنبال `"chat":{"id":` بگردید. برای اکانت شخصی معمولاً عددی مثبت است، مثلاً `123456789`.  
   این همان **`adminnumber`** در `config.php` است.

**نکته:** اگر `getUpdates` خالی بود، یک پیام دیگر به ربات بفرستید و دوباره همان لینک را باز کنید.

### گام ۳: یوزرنیم ربات را یادداشت کنید

بدون `@`، فقط نام کاربری، مثلاً `MyShopVPN_bot`.  
در `config.php` فیلد **`usernamebot`** است.

## بخش ۲ — نصب روی سرور (دو روش)

### روش الف — نصب خودکار با اسکریپت رسمی (ساده‌تر)

روی سرور Ubuntu به‌عنوان root:

```bash
export GITHUB_PAT="ghp_your_token_with_repo_scope"

curl -fsSL \
  -H "Authorization: token ${GITHUB_PAT}" \
  -H "Accept: application/vnd.github.raw" \
  "https://raw.githubusercontent.com/Recoba86/Bold-Connection/main/install.sh" \
  -o /tmp/bold-install.sh

sudo bash /tmp/bold-install.sh
```

گزینه **۱ (Install)** را بزنید. اسکریپت از شما می‌پرسد:

- **دامنه** (مثلاً `bot.example.com`)
- **Bot Token**
- **Chat ID** (ادمین)
- **usernamebot**

سپس خودش:

- SSL و Apache را تنظیم می‌کند
- فایل‌ها را فقط از **Recoba86/Bold-Connection** کلون می‌کند (نه Mirza Pro)
- مسیر پیش‌فرض: `/var/www/html/bold-connection`
- `config.php` می‌سازد
- **Webhook** را با `secret_token` ثبت می‌کند
- یک بار `table.php` را صدا می‌زند (ساخت جداول دیتابیس)

> اگر کد **همین مخزن 0.1.5** را می‌خواهید، فایل‌ها را دستی آپلود کنید (روش ب).

### روش ب — آپلود همین پروژه (دستی)

1. کل پوشه را روی سرور بگذارید، مثلاً:

   ```
   /var/www/html/bold-connection/
   ```

2. مالکیت را به وب‌سرور بدهید:

   ```bash
   sudo chown -R www-data:www-data /var/www/html/bold-connection
   sudo chmod -R 755 /var/www/html/bold-connection
   ```

3. VirtualHost Apache/Nginx طوری باشد که **DocumentRoot** همان پوشه باشد و `https://bot.example.com/index.php` در مرورگر خطای PHP ندهد (یا 200 خالی برگرداند).

4. دیتابیس MySQL بسازید و `config.php` را پر کنید (نمونه در بخش ۳).

5. یک بار در مرورگر باز کنید:

   ```
   https://bot.example.com/table.php
   ```

   این جداول را می‌سازد و در انتها webhook را هم تنظیم می‌کند (اگر `domainhosts` درست باشد).

## بخش ۳ — پر کردن `config.php`

فایل: `config.php` در ریشهٔ پروژه.

| متغیر | مقدار | از کجا |
|--------|--------|--------|
| `$dbhost` | معمولاً `localhost` | سرور MySQL |
| `$dbname` | نام دیتابیس | خودتان ساختید |
| `$usernamedb` / `$passworddb` | کاربر DB | خودتان |
| `$APIKEY` | توکن BotFather | بخش ۱ |
| `$adminnumber` | Chat ID شما | بخش ۱ |
| `$domainhosts` | فقط دامنه، **بدون** `https://` و بدون `/` در انتها | مثال: `bot.example.com` |
| `$usernamebot` | یوزرنیم بدون @ | مثال: `MyShopVPN_bot` |

### امنیت (نسخهٔ فعلی شما — حتماً در پروداکشن)

```php
$allow_self_signed_certs = false;

// یک رشته تصادفی طولانی — باید با setWebhook یکی باشد
$telegram_webhook_secret = 'رشته_تصادفی_قوی_حداقل_۳۲_کاراکتر';

// برای webhook پنل (card / marzban و …) — جدا از رمز ادمین
$payment_webhook_key = 'رشته_تصادفی_دیگر_قوی';
```

تولید رشته تصادفی روی سرور:

```bash
openssl rand -hex 32
```

## بخش ۴ — ثبت Webhook (قلب اتصال تلگرام به سرور)

تلگرام هر پیام کاربر را به آدرس زیر **POST** می‌کند:

```
https://YOUR_DOMAIN/index.php
```

### ثبت با secret (توصیه‌شده — هم‌خوان با `index.php` شما)

```bash
DOMAIN="bot.example.com"
TOKEN="7123456789:AAH..."
SECRET="همان_مقدار_telegram_webhook_secret_در_config"

curl -s "https://api.telegram.org/bot${TOKEN}/setWebhook" \
  -d "url=https://${DOMAIN}/index.php" \
  -d "secret_token=${SECRET}"
```

پاسخ باید `"ok":true` باشد.

### بررسی وضعیت webhook

```bash
curl -s "https://api.telegram.org/bot${TOKEN}/getWebhookInfo"
```

چیزهایی که باید ببینید:

- `"url": "https://bot.example.com/index.php"`
- `"last_error_message"` خالی یا قدیمی نباشد
- اگر خطای SSL یا 403/500 دیدید، لاگ Apache/PHP را چک کنید

### حذف webhook (فقط برای عیب‌یابی)

```bash
curl -s "https://api.telegram.org/bot${TOKEN}/deleteWebhook"
```

بعد دوباره `setWebhook` بزنید.

### اگر از Long Polling استفاده کرده بودید

قبل از webhook حتماً **`deleteWebhook`** بزنید؛ تلگرام همزمان polling و webhook را برای یک ربات نگه نمی‌دارد.

## بخش ۵ — اولین تست در تلگرام

1. به ربات بروید و **`/start`** بزنید.
2. اگر جواب آمد → webhook و دیتابیس درست است.
3. اگر سکوت بود:
   - `getWebhookInfo` → `last_error_message`
   - لاگ: `/var/log/apache2/...-error.log`
   - آیا `table.php` یک بار اجرا شده؟
   - آیا `$telegram_webhook_secret` با `secret_token` در `setWebhook` **دقیقاً یکی** است؟

4. برای پنل ادمین ربات، همان اکانتی که `$adminnumber` است باید `/start` زده باشد؛ بعد از نصب منوهای مدیریت در همان چت ظاهر می‌شود.

---

## بخش ۶ — اتصال **پنل VPN صنعتی** به ربات (بعد از تلگرام)

این بخش در تلگرام نیست؛ در **پنل Marzban / مشابه** است، ولی بدون آن خرید خودکار ساخته نمی‌شود.

1. در ربات (منوی ادمین) **پنل** را اضافه کنید: آدرس API، یوزر/پسورد پنل، نوع پنل.
2. برای رویدادهای پنل (مثلاً ساخت کاربر، انقضا) آدرس webhook ربات:

   ```
   https://YOUR_DOMAIN/webhooks.php
   ```

3. هدر امنیتی (در نسخهٔ فعلی):

   - نام هدر: **`X-Webhook-Secret`**
   - مقدار: **Base64** از همان `$payment_webhook_key`  
     (یا اگر خالی گذاشتید، سیستم قدیمی از رمز ادمین در DB استفاده می‌کند — توصیه نمی‌شود.)

   تولید مقدار Base64 روی سرور:

   ```bash
   echo -n 'همان_payment_webhook_key' | base64
   ```

4. درگاه‌های پرداخت (زرین‌پال، کارت، …) callback URL جدا دارند؛ معمولاً زیر `payment/` — از منوی تنظیمات ربات کپی می‌شود.

## بخش ۷ — Cron (کارهای پس‌زمینه)

ربات برای انقضا، اعلان، کارت خودکار، بکاپ و … به **cron** روی سرور نیاز دارد.

- با ورود به **پنل ادمین** (`admin.php` از طریق خود ربات)، تابع `activecron()` سعی می‌کند jobها را در crontab سرور اضافه کند.
- اگر سرور اجازهٔ `crontab` ندهد، دستی این‌ها را اضافه کنید (دامنه را عوض کنید):

```cron
*/15 * * * * curl -s https://bot.example.com/cronbot/statusday.php
*/1  * * * * curl -s https://bot.example.com/cronbot/croncard.php
*/1  * * * * curl -s https://bot.example.com/cronbot/NoticationsService.php
*/5  * * * * curl -s https://bot.example.com/cronbot/payment_expire.php
*/1  * * * * curl -s https://bot.example.com/cronbot/sendmessage.php
```

لیست کامل در `function.php` تابع `activecron()` است.

## عیب‌یابی سریع

| مشکل | احتمال |
|------|--------|
| ربات جواب نمی‌دهد | webhook اشتباه، SSL، یا `secret_token` با config فرق دارد |
| `401` در لاگ | `$telegram_webhook_secret` با setWebhook هم‌خوان نیست |
| خطای دیتابیس | `table.php` اجرا نشده یا اطلاعات DB در config غلط است |
| خرید انجام می‌شود ولی کانفیگ نمی‌سازد | پنل در ربات تنظیم نشده یا `webhooks.php` / API پنل درست نیست |
| پرداخت تأیید نمی‌شود | cron یا URL درگاه در `payment/` |

---

## خلاصهٔ یک خطی

**در تلگرام:** BotFather → توکن + Chat ID → `/start`  
**روی سرور:** کد + DB + `config.php` + SSL + `setWebhook` → `table.php`  
**در پنل VPN:** آدرس API + `webhooks.php`  
**بعد:** `/start` و تنظیم محصولات از ادمین ربات.
